# Conventions

## Главные принципы
- Один файл = одна тема.
- Сначала фиксировать `fact`, потом `decision`, потом `assumption`, потом `open question`.
- Каноническая правда не должна содержать шум из одного разговора.
- Если сигнал неустойчивый, он идет в session log, а не в founder-memory.

## Язык
- Основной язык содержания: русский.
- Имена файлов, slug'и и часть метаданных могут быть на английском.

## Формат
- Все основные документы — Markdown.
- Для важных файлов желательно использовать короткий frontmatter.

## Статусы
- `active` — актуальный документ.
- `draft` — черновик.
- `superseded` — заменен новым документом.
- `archived` — исторический документ, не читать по умолчанию.

## Источник правды
- Для проекта: `02_ventures/<name>/canonical/`
- Для founder-memory: `01_founder/`
- Для решений: `06_decisions/decision-log.md`
- Для дневной динамики: `07_sessions/`

## Правила обновления
- Команда `обновись` обновляет проект, session log и founder-memory при наличии нового устойчивого паттерна.
- Команда не должна молча переписывать каноническую правду без явного нового решения.
- Перед большим апдейтом система должна коротко написать, что именно обновит.
