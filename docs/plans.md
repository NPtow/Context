# План: LLM-first memory system для `Context`

## Кратко
- Цель: построить долгоживущую memory-систему, удобную прежде всего для LLM, а не для ручного чтения.
- Ключевой интерфейс: короткие команды на естественном языке, в первую очередь `обновись`.
- Система должна хранить одновременно: актуальное состояние, session log, память о Никите, историю проектов и архив.
- Репозиторий пустой, поэтому можно сразу заложить правильную архитектуру без миграций.

## Scope
- Спроектировать и зафиксировать структуру репозитория под долгую LLM-память.
- Описать протоколы:
  - update protocol;
  - retrieval protocol;
  - temporal protocol.
- Зафиксировать набор команд на естественном языке и правила их интерпретации.
- Подготовить seed-файлы для:
  - активного проекта `referalka`;
  - долгой памяти о Никите;
  - session log за текущий день;
  - доменного знания по recruiting / Jack & Jill.
- Настроить индексные файлы так, чтобы LLM могла быстро определить, что читать первым и что является канонической правдой.

## Out of scope для первой итерации
- Автоматическая индексация через embeddings / векторную БД.
- Сложная генерация summary на каждый файл.
- Интеграции с внешними системами кроме git.
- Автоматическое обновление без явной команды пользователя.

## Предварительные решения
- Команды задаются на естественном языке.
- Команда `обновись` по умолчанию:
  - обновляет активный проект;
  - добавляет session log;
  - обновляет память о Никите, если появились новые устойчивые паттерны.
- Активный проект определяется по текущему диалогу; если уверенность низкая, допускается один уточняющий вопрос.
- Актуальный контекст должен быть доступен без специального флоу: если проект активен, модель должна быстро выйти на его текущее состояние.

## Assumptions
- Хранилище будет использоваться долго, даже если отдельные проекты умрут.
- Пользователь в основном обращается к репозиторию через LLM, а не напрямую.
- Для экономии токенов нужен явный индекс и разделение на `active`, `memory`, `sessions`, `archive`.
- Документы можно хранить в mixed-format: русский основной текст + английские имена файлов и метаданные.
