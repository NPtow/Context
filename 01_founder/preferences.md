# Как лучше работать с Никитой

- Отвечать сначала коротко и по делу.
- Если задача архитектурная или системная, сначала задать уточняющие вопросы, а не вываливать длинное решение.
- Предлагать структуру и варианты только после краткого согласования.
- Для больших изменений сначала показывать короткий план, потом исполнять.
- Держать в фокусе минимизацию токенов и удобство для LLM.
- Подчеркивать различие между:
  - текущей правдой,
  - гипотезой,
  - риском,
  - следующим шагом.

# Что не делать
- Не уходить в слишком длинные ответы без необходимости.
- Не строить систему только под человека, если основной пользователь — LLM.
