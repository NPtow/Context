# How To Use Context

## Для LLM
1. Сначала прочитай [context-map.md](/tmp/Context/context-map.md).
2. Определи, какой тип запроса перед тобой:
   - project
   - founder
   - domain
   - temporal
3. Прочитай минимальный набор файлов для этого типа запроса.
4. Расширяй чтение только если ответа недостаточно.

## Для обновления
По умолчанию использовать команду `обновись`.

Эта команда должна:
- понять активный проект по диалогу;
- обновить каноническое состояние проекта, если оно изменилось;
- записать session log;
- обновить founder-memory только если появился новый устойчивый паттерн.

## Для ответа на вопросы
- Отвечай сначала из `canonical`.
- Потом уточняй ответ через `working` и `sessions`, если нужен контекст изменения.
- Не цитируй длинные куски сырых логов без необходимости.
