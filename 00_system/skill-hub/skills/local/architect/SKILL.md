# architect

Скилл для безопасной систематизации файловой системы. Принципы: **никогда не удалять**, только перемещать и архивировать. Каждое действие логируется в манифест.

## Когда использовать

- Пользователь просит разобрать папки: "систематизируй", "разбери", "почисти"
- Пользователь хочет архивировать старые файлы
- Нужно понять что где лежит и предложить структуру
- Папки Downloads / Desktop переполнены

## Целевая структура папок

```
~/Projects/          — активные проекты (код, прод-работа)
  └── <project-slug>/

~/Work/              — рабочие документы
  ├── briefs/        — ТЗ, брифы
  ├── surveys/       — опросы, JSON-экспорты
  ├── contracts/     — договоры, соглашения
  ├── presentations/ — питчи, презентации
  └── reports/       — аналитика, отчёты

~/Media/             — медиа-контент
  ├── recordings/    — аудио/видео встреч и созвонов
  ├── reels/         — контент для соцсетей
  └── screenshots/   — скриншоты экрана

~/Archive/           — архив (НИКОГДА не удаляется)
  ├── YYYY-QN/       — по кварталам
  └── misc/          — всё что не удалось классифицировать
```

## Режимы работы

### 1. `scan` — анализ (только чтение, ничего не меняет)

Действия:
1. Сканировать указанные папки (по умолчанию: Desktop, Downloads)
2. Группировать файлы по типам: docs, media, images, code, archives, misc
3. Выделить активные проекты (папки с git, package.json, README)
4. Выделить кандидатов на архив (файлы старше N дней без изменений)
5. Вывести краткий отчёт: сколько файлов, сколько GB, топ-категории

Вывод: таблица + список рекомендаций, без изменений файлов.

### 2. `propose` — предложить план реорганизации

На основе scan предложить конкретные перемещения:
- Какие файлы → куда переместить
- Что архивировать (с указанием целевой папки в Archive/)
- Что оставить на месте (активные проекты)

Формат предложения: список `mv` команд в файл `~/architect-plan.sh` (не выполнять автоматически без явного запроса).

### 3. `apply` — применить план

1. Прочитать `~/architect-plan.sh`
2. Создать манифест `~/architect-manifest-YYYY-MM-DD.json` с записью каждого перемещения
3. Создать целевые папки если они не существуют
4. Выполнить перемещения через `mv` (не `cp`, не `rm`)
5. Записать результат в манифест
6. Вывести итог: сколько файлов перемещено, сколько пропущено

### 4. `archive` — архивировать старое

Параметры:
- `--older-than N` — файлы старше N дней (default: 90)
- `--folder PATH` — конкретная папка (default: ~/Downloads)
- `--target PATH` — куда архивировать (default: ~/Archive/YYYY-QN/)

Алгоритм:
1. Найти файлы по критерию
2. Показать список (без выполнения)
3. Создать манифест
4. Переместить в Archive/ сохраняя относительные пути

### 5. `status` — текущее состояние

Показать:
- Размер ключевых папок (Desktop, Downloads, Archive)
- Количество файлов по категориям
- Дату последней реорганизации (из манифеста)
- Кандидаты на архивирование (если есть)

## Правила безопасности

1. **Никогда не использовать `rm`** — только `mv`
2. **Всегда создавать манифест** перед любым перемещением
3. **Проверять существование целевой папки** перед mv
4. **Не трогать системные папки**: Library, .Trash, скрытые папки (начинаются с `.`)
5. **Не трогать активные git-репозитории** (папки с `.git`) — только документировать
6. **Не трогать node_modules, .next, .vercel** и другие build-артефакты
7. **Дубликаты**: если файл с таким именем уже существует в цели — добавить суффикс `_archived_YYYYMMDD`
8. **Манифест сохранять** в `~/architect-manifests/` для возможности rollback

## Rollback

Манифест содержит записи `{from, to, timestamp}`. Для отката:
```bash
# Читаем манифест и выполняем обратные mv
# from ↔ to меняются местами
```

## Соглашения по именованию

- План: `~/architect-plan.sh`
- Проекты: lowercase-slug (например `jjfr`, `swipestyle`)
