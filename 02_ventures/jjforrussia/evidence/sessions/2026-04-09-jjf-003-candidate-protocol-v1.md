# Что произошло

- Для `JJF-003` собран полный пакет по интервью с кандидатами, а не только questions-only draft.
- Пакет намеренно разделен на несколько артефактов, чтобы не смешивать сценарий разговора, поля фиксации, шаблон заметки и отдельную анкету предпочтений.

# Что теперь входит в JJF-003

- полный протокол интервью;
- отдельные поля для фиксации после разговора;
- шаблон заметки по интервью;
- короткая анкета предпочтений;
- мастер-файл `candidate-interview-pack-v1.md`, который собирает пакет в один операционный документ.

# Что это уточнило

- questions-only draft больше не считается достаточным результатом по `JJF-003`;
- предпочтения кандидата теперь вынесены из основного интервью, чтобы не размывать разговор про боль поиска и полезность формата;
- в самом протоколе появился явный выбор решения после интервью:
  - вести дальше;
  - отложить;
  - не вести;
- пакет прямо опирается на `PM role taxonomy v1`, чтобы статус кандидата ставился одинаково.

# Как это связано со skill по расшифровке

- `meeting-transcriber` не заменяет протокол;
- протокол описывает, как проводить интервью и что из него фиксировать;
- skill расшифровки нужен после созвона, если есть запись и её надо сохранить в `10_meetings` и связать с проектом.
