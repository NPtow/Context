# Summary

- Для `jjforrussia` оформлен reusable Telegram sourcing surface как отдельный skill `telegram-hiring-contact-sourcing` в `skill-hub` и как локальная установленная копия.
- Для этого surface собрана отдельная инструкция по Telethon-подключению, session lifecycle, reset/re-auth и output contract для `Telegram -> vacancy/contact extraction -> CSV`.
- В ходе работы отдельно исследована проблема создания Telegram API app на `my.telegram.org`, включая ошибки `ERROR` и `[object Object]`.
- Позже пользователь принес серый operational path с покупным аккаунтом, прокси и замороженной сессией; этот путь явно не принят в канонический workflow и не должен считаться нормой для skill.
- Cloud-копия skill в `Context` выровнена обратно к безопасной модели: `Telethon + own account + local session file + Telegram-only + exact-only`.

# Что изменилось

- `skill-hub` теперь содержит не только mirrored local skills, но и первый venture-relevant cloud skill для Telegram sourcing ops.
- Для `jjforrussia` появился reusable operator surface, который может поддержать задачу поиска первых target contacts без разрастания продукта в серый scraping stack.
- В проектной truth зафиксировано, что допустимый sourcing workflow должен идти через собственный Telegram-аккаунт пользователя и evidence-backed extraction.
- Отдельно собран source-bearing readout по Telegram app creation troubleshooting, но community workarounds не приняты как каноническая truth.
